# Memory Image V1 Notes

这份笔记用于固定当前已经达成共识的 `memory_image` 方案，服务于后续实现、实验设计与论文撰写。

当前目标不是把 `me_block` 做成一个复杂的端到端时序模块，而是先做一个可解释、可离线训练、可与当前 ACT 主干低耦合集成的 V1 系统。

## 1. 核心目标

### 1.1 要解决的问题

我们想解决的根本问题不是“摄像头前出现遮挡”本身，而是：

- ACT 对历史关键信息的持续表征能力不足
- 当前帧一旦看不到目标，模型缺少稳定的外部记忆
- 失败场景不仅包括外部遮挡，还包括：
  - 机械臂自身遮挡目标
  - 场景中临时出现的其他障碍
  - 目标或关键标记短时不可见

因此，`memory_image` 的目标不应被狭义理解为“去遮挡图像”，而应理解为：

- 当前时刻的历史补偿图
- 当前时刻的任务场景记忆图
- 一个供 ACT 读取的可解释外部视觉记忆

### 1.2 为什么不做端到端长时序 memory

当前不推荐直接把记忆模块设计成“对整条轨迹做反向传播的时序网络”，原因包括：

- 轨迹长度长，BPTT 显存和时间成本高
- 训练不稳定，调试困难
- 当前数据加载方式并不天然适配长序列端到端训练
- 工程上很难先做出一个可稳定验证的版本

因此，V1 采用的总路线是：

1. 先训练或构造一个独立的 `memory_image` 生成系统
2. 该系统逐帧递推生成 `memory_image`
3. 训练好后冻结，或直接离线跑完整个数据集
4. 在 ACT 训练阶段，把生成好的 `memory_image_four_channel` 当作“虚拟摄像头”输入

这样做的优点：

- 避免长序列 BPTT
- 保留可解释性
- 与当前 ACT 主干兼容性高
- 便于单独分析记忆模块质量

## 2. V1 的总体设计

### 2.1 总体思想

V1 不直接让网络输出最终 `memory_image`，而是采用“两阶段”思想：

1. 轻量分割网络输出每一类对象的重要性概率图
2. 用显式、可解释的递推规则更新记忆状态，生成 `memory_image`

也就是说，网络负责回答：

- 当前帧哪些像素属于关键对象

而记忆系统负责回答：

- 当前帧是否应该用这些像素覆盖旧记忆
- 哪些旧记忆应该保留
- 最终哪些像素应该暴露给 ACT

### 2.2 V1 不使用 `qpos`

虽然 `qpos` 对机械臂状态有帮助，但 V1 先不引入 `qpos`，原因是：

- 会增加训练与接口复杂度
- 当前更想先把纯视觉记忆逻辑跑通
- ACT 主干本身已经使用 `qpos`

因此，V1 的记忆模块输入仅考虑视觉输入。

### 2.3 V1 输出仍为 4 通道图像

当前 ACT 主干已经兼容：

- `image`
- `memory_image`

因此 V1 继续保持：

- `memory_image` 与当前 `four_channel` 同分辨率
- 4 通道格式不变
- 逐帧一一对应

这样主线几乎不需要结构性改动。

## 3. Importance-Only 记忆更新公式

### 3.1 每类输出一个概率图

轻量分割网络对当前帧输出 `K` 个概率图：

- `p_1`
- `p_2`
- `...`
- `p_K`

每个 `p_c` 表示“该像素属于第 `c` 类关键对象”的概率。

V1 的初始类别建议：

- `target`
- `goal`
- `arm`

后续允许扩展：

- `target_2`
- `obstacle`
- `tool`
- 其他任务相关对象

### 3.2 类别权重配置化

每一类都配置一个权重 `w_c`，并且在配置里先归一化到和为 `1`。

V1 的直觉设定可以是：

- `target` 权重最高
- `goal` 次之
- `arm` 很小，但不为 `0`

设计原则：

- 机械臂在无遮挡情况下仍然可以进入记忆
- 机械臂与旧目标记忆冲突时，由于权重低，会自然让步
- 后续加类时，只需要增加类别和权重，不需要改公式

### 3.3 递推公式

记忆系统内部维护两个状态：

- `M_t`：记忆图像
- `S_t`：记忆置信图

当前帧分割网络输出 `p_c` 后，先计算：

```text
Q_t = sum_c w_c * p_c
```

其中：

- `sum_c w_c = 1`
- `Q_t` 取值自然落在 `[0, 1]`

然后进行递推更新：

```text
S^-_t = lambda * S_{t-1}

W_t = Q_t > S^-_t + tau_up

M_t = where(W_t, I_t, M_{t-1})
S_t = where(W_t, Q_t, S^-_t)

E_t = S_t > tau_out
memory_image_t = M_t * E_t
```

其中：

- `I_t` 为当前 4 通道输入图像
- `lambda` 为记忆衰减系数
- `tau_up` 为覆盖旧记忆所需的增量门槛
- `tau_out` 为输出给 ACT 的显示门槛

### 3.4 为什么要有 `tau_up` 和 `tau_out`

这两个阈值作用不同：

- `tau_up` 控制“新观测是否足够强，值得覆盖旧记忆”
- `tau_out` 控制“当前记忆是否仍然足够可靠，可以暴露给 ACT”

因此原则上：

- 写入要保守
- 输出可以略宽松

这样可以减少记忆闪烁，提升时序稳定性。

### 3.5 稀疏图输出

V1 输出给 ACT 的 `memory_image` 采用稀疏图，而不是全景平均图。

原因：

- ACT 已经能看到当前帧
- `memory_image` 的职责是补充“当前不容易看到、但历史中关键”的内容
- 稀疏图更容易解释
- depth 不适合做平均融合

因此 V1 采用：

- RGB 也走稀疏输出
- depth 只在像素被选中时覆盖
- 不做深度加权平均

## 4. 为什么采用统一 segmentation 路线

### 4.1 拒绝“每类一个方法”

虽然不同对象可以用不同启发式方法单独做标注，但这不够优雅，也不利于扩展。

当前更倾向于：

- 所有关键对象统一用同一类方法识别
- 所有类别统一由一个多类分割模型输出
- 后续扩类只增加类别定义和权重，不改主流程

这更符合：

- 工程统一性
- 任务可扩展性
- 方法描述的整洁性

### 4.2 统一方案

当前推荐的统一方案是：

1. 离线伪标注统一使用“开放词汇检测/分割”链路
2. 在线统一使用一个轻量多类分割网络
3. 分割结果统一进入显式记忆递推规则

即：

- Teacher：大模型，负责离线伪标注
- Student：小模型，负责在线推理

### 4.3 推荐的 teacher/student 组合

推荐组合：

- Teacher：
  - `GroundingDINO`
  - `MobileSAM` 或 `Grounded-Segment-Anything`
- Student：
  - `LR-ASPP + MobileNetV3-Large`

理由：

- Teacher 适合开放词汇、少量人工修正、快速扩类
- Student 足够轻量，便于在线推理
- 多类输出天然匹配 `p_1 ... p_K`
- 与现有 PyTorch 工程兼容性好

## 5. Importance Map 的定义

### 5.1 语义定义

`importance map` 表示：

- 该像素属于关键对象的概率

而不是：

- 前景显著性
- 遮挡概率
- 单纯的视觉注意力

也就是说，importance 是对象级、任务相关的。

### 5.2 初始类别

V1 初始类别：

- `target`
- `goal`
- `arm`

说明：

- `target`：核心操作对象
- `goal`：目标放置点，当前是黑色小盖子
- `arm`：机械臂本体或末端执行器

### 5.3 可扩展性要求

类别定义必须可扩展，后续新增对象时应满足：

- 仅新增一个类别名
- 新增对应文本 prompt
- Student 分割头增加一个输出通道
- 权重表增加一项

而不需要重写记忆更新逻辑。

## 6. 伪标注数据生产方案

### 6.1 基本思想

不直接依赖人工逐帧标注，而是采用：

- 开放词汇 teacher 先生成伪标签
- 人工少量修正
- 再训练轻量 student

这条路线兼顾：

- 通用性
- 扩类便利性
- 标注成本
- 推理效率

### 6.2 伪标注流程

对每个类别，使用统一 prompt 机制：

- `target`
- `goal`
- `arm`

典型流程：

1. 用 `GroundingDINO` 根据文本 prompt 找框
2. 用 `MobileSAM` 或 `Grounded-Segment-Anything` 细化 mask
3. 得到该类 mask
4. 合并为多类标签图

### 6.3 人工修正

允许人工修正少量关键帧，数量级控制在约 `100` 张左右。

这些样本主要用于：

- 调整 prompt
- 校正小目标 mask
- 修正遮挡边界
- 提高 student 的初始监督质量

### 6.4 关于 `goal` 的特殊性

`goal` 目前是一个黑色小盖子，视觉上较小。

虽然不打算为它单独设计另一套方法，但训练时应注意：

- 小目标可能召回不足
- 伪标注可能需要更高分辨率 crop
- student 训练时可以对小目标类别提高 loss 权重

这仍然属于统一 segmentation 路线内部的工程增强，而不是换方法。

## 7. Student 网络的推荐形式

### 7.1 轻量多类分割

V1 推荐 student 为轻量多类分割网络，直接输出：

- `p_target`
- `p_goal`
- `p_arm`

以及后续可扩展类别。

### 7.2 推荐模型

首选：

- `torchvision.models.segmentation.lraspp_mobilenet_v3_large`

优点：

- 官方实现
- 与当前 PyTorch 环境兼容
- 参数量和算力较轻
- 容易改成多类输出

如果后续效果不足，再考虑升级到更强但更重的结构。

## 8. 记忆状态与输出形式

### 8.1 内部状态

V1 内部保存：

- `M_t`：形状与 `I_t` 相同的 4 通道记忆图
- `S_t`：单通道置信图

`S_t` 只服务于记忆系统内部，不直接输入 ACT。

### 8.2 对 ACT 的输出

ACT 只接收：

- `memory_image_t`

不接收：

- `S_t`
- `Q_t`
- 分类概率图

这样当前 ACT 主线最小改动即可继续工作。

### 8.3 为什么不把 `S_t` 喂给 ACT

原因：

- 增加主线耦合度
- ACT 当前接口无需承担记忆内部控制信息
- V1 更希望先验证“图像式外部记忆”本身是否有增益

因此：

- `S_t` 用于更新、调试、可视化
- `memory_image_t` 用于 ACT 训练和推理

## 9. 关于遮挡数据的地位

### 9.1 遮挡不是最终目标，而是重要验证手段

遮挡模拟数据不是整个记忆模块的唯一意义，但它仍然非常重要，因为它可以：

- 提供一种可控验证方式
- 帮助测试历史补偿机制是否有效
- 为 teacher/student 的训练和评估提供监督场景

因此，遮挡实验更适合被理解为：

- 记忆模块的一种受控验证
- 同时也是一种训练增强

而不是记忆模块存在的全部理由。

### 9.2 遮挡数据仍然建议保留

当前仍然建议后续实现：

- `task_obst_*`
- `occlusion_masks/`
- `occlusion_meta.json`

因为这对于后续：

- 控制实验
- 结果分析
- 论文展示

都很有帮助。

## 10. 建议的实现顺序

### 阶段一：固定方案与配置

先固定以下设计：

- 类别列表
- 类别权重
- prompt 配置
- `lambda`
- `tau_up`
- `tau_out`

并写入配置文件或单独的记忆模块配置。

### 阶段二：伪标注数据生成

实现统一伪标注流程：

- 输入：原始 `four_channel` 数据
- 输出：多类 mask 标签
- 支持人工修正

### 阶段三：训练轻量 student

训练一个多类分割 student，输出 `p_1 ... p_K`。

### 阶段四：实现递推记忆生成器

实现逐帧更新逻辑，输入：

- 当前帧 `I_t`
- 当前 student 输出的多类概率图
- 上一时刻 `M_{t-1}`
- 上一时刻 `S_{t-1}`

输出：

- `M_t`
- `S_t`
- `memory_image_t`

### 阶段五：离线为训练集生成 `memory_image_four_channel`

对整个 ACT 数据集逐帧递推生成：

- `memory_image_four_channel`

并写入现有数据目录结构。

### 阶段六：冻结后接入 ACT

ACT 训练阶段不再训练记忆模块，而是直接读取离线生成的 `memory_image`。

## 11. 论文写作可用的表述方向

当前方案在论文中可以表述为：

- 一个对象级的可解释外部视觉记忆
- 通过轻量多类分割估计关键对象重要区域
- 通过显式递推规则维护历史视觉记忆
- 最终以稀疏 4 通道记忆图的形式输入 ACT

和常见黑盒时序模块相比，这个方案的特点是：

- 可解释
- 低耦合
- 易分析
- 避免长序列 BPTT
- 便于替换和扩类

## 12. 当前已确定的点

- V1 不引入 `qpos`
- 使用统一 segmentation 路线，不为不同类别设计不同识别方法
- 类别权重在配置里归一化到和为 `1`
- 使用 `Q_t = sum_c w_c * p_c`
- 采用内部 `M_t / S_t` 递推
- 输出给 ACT 的是稀疏 `memory_image`
- `S_t` 不进入 ACT 主线
- teacher/student 路线优于纯零样本在线推理

## 13. 当前待定问题

这些问题后续实现前还需要进一步敲定：

- teacher 的具体 prompt 如何写
- student 训练时各类别 loss 权重如何设置
- `target / goal / arm` 的初始权重具体取值
- `lambda / tau_up / tau_out` 的初始默认值
- 稀疏图是否需要对 `target` 和 `goal` 做轻微膨胀保留少量上下文
- 伪标注阶段是否对小目标区域使用 crop / tile 提高质量

## 14. 当前不采用的路线

以下路线暂时不作为 V1：

- 整轨迹端到端时序训练
- 直接把 `me_block` 接进 ACT 联合训练
- 不可解释的黑盒 feature memory
- 每个类别分别使用不同识别算法
- 直接在线运行大模型作为最终部署方案

## 15. 一句话总结

V1 的本质是：

先用统一的多类分割模型找出当前帧中“哪些像素属于关键对象”，再用显式、可解释、可递推的规则，把这些对象级信息写入一个稀疏的 4 通道视觉记忆图中，最后把该记忆图作为 ACT 的附加视觉输入。
